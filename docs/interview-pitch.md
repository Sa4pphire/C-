# 两分钟项目介绍

我用 C++17 实现了一个轻量任务管理后端 TaskHub。它提供创建、查询、更新和删除任务的 HTTP API，并用 SQLite 保证重启后数据仍然存在。

架构上，我把 HTTP 路由、业务校验和数据访问拆成三层。路由层只负责 HTTP 和 JSON；`TaskService` 负责标题、描述和状态校验；`SqliteTaskRepository` 负责 SQL。这样业务层依赖仓库接口而不是具体数据库，后续可以更换为 MySQL 实现。

在资源管理上，数据库连接和预编译 SQL 语句都通过 RAII 自动释放，避免了 C 风格手动释放遗漏的问题。服务每个连接在独立线程中处理；由于 SQLite 使用共享连接，我用 `std::mutex` 和 `std::lock_guard` 保证数据库操作串行化，并加入并发创建任务的测试来验证不会丢数据。

项目通过 CMake 构建、CTest 执行 23 个测试，覆盖正常流程、参数错误、任务不存在、JSON 解析失败和并发写入。当前版本的目标是教学与展示；下一步我会把一请求一线程替换成线程池，并接入 MySQL、Redis 和更成熟的网络框架。
